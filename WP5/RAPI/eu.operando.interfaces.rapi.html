Regulations

Up 
post /regulations
Add a new regulation to the system. (regulationsPost)
Called by a regulator to ensure that the platform becomes compliant with the new regulation. The regulation that was saved is returned, along with an identifer which should be used to refer to it in later communication.

Request body

regulation body Regulation body (required)
Body Parameter —

Request headers

Return type

PrivacyRegulation

Example data

Content-Type: application/json
{
  "reason" : "aeiou",
  "private_information_type" : "aeiou",
  "required_consent" : "aeiou",
  "action" : "aeiou",
  "legislation_sector" : "aeiou",
  "reg_id" : "aeiou"
}

Produces

This API call produces the following media types according to the Accept request header; the media type will be conveyed by the Content-Type response header.
  • application/json

Responses

201

The regulation was successfully created, and the platform has been made compliant with it. PrivacyRegulation

401

The user is not authenticated with the OPERANDO system. Check that the service ticket provided by the authentication service is correctly included in the message body.

403

The user is authenticated with the OPERANDO system, but is not allowed to perform the requested action.

Reports

Up 
get /osps/{osp-id}/compliance-report
Get a report. (ospsOspIdComplianceReportGet)
Called by a regulator to obtain a compliance report relating to the specified OSP.

Path parameters

osp-id (required)
Path Parameter — the unique identifier of an online service provider

Request headers

Return type

ComplianceReport

Example data

Content-Type: application/json
{
  "privacypolicy" : {
    "policies" : [ {
      "reason" : "aeiou",
      "datauser" : "aeiou",
      "reasonid" : "aeiou",
      "datatype" : "aeiou",
      "datasubjecttype" : "aeiou"
    } ],
    "osp_policy_id" : "aeiou"
  }
}

Produces

This API call produces the following media types according to the Accept request header; the media type will be conveyed by the Content-Type response header.
  • application/json

Responses

200

The request was successful. The report is returned in the response body in the requested format. ComplianceReport

401

The user is not authenticated with the OPERANDO system. Check that the service ticket provided by the authentication service is correctly included in the message body.

403

The user is authenticated with the OPERANDO system, but is not allowed to perform the requested action.

AccessReason - Up

reasonid (optional)
String An identifier within this privacy policy e.g osp_reason1. This is not globally unique and is defined by the OSP. However, OSP must ensure that identifiers are unique within the policy list.
datauser (optional)
String Who is using the data.
datasubjecttype (optional)
String What is the type or group of the user this private data concerns e.g. patient, helper etc.
datatype (optional)
String What is the type of data this refers to e.g. e-mail address, medical record etc.
reason (optional)
String Usage of this information if for what purpose e.g. marketing, healthcare delivery.

PrivacyRegulation - Up

A privacy rule that reflects a given privacy legislation as described by a particular set of laws in a given jurisdiction.
reg_id (optional)
String
legislation_sector
String The identifier of the set of legislation rules this rule belongs to e.g. the UK data protection act.
reason (optional)
String The purpose for performing an action on data e.g. medical care, marketing, admin
private_information_type (optional)
String The type of private information; e.g. is it information that identifies the user (e.g. id number)? is it location information about the user? Is it about their activities?
action (optional)
String The action being carried out on the data
required_consent (optional)
String The type of consent that must be followed
