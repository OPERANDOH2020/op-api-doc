post /regulations
Add a new regulation to the system. (regulationsPost)
Called by a regulator to ensure that the platform becomes compliant with the new regulation. The regulation that was saved is returned, along with an identifer which should be used to refer to it in later communication.

Request body

regulation body (required)
Body Parameter —

Return type

PrivacyRegulation

Produces

This API call produces the following media types according to the Accept request header; the media type will be conveyed by the Content-Type response header.
  • application/json

Responses

201

The regulation was successfully created, and the platform has been made compliant with it. PrivacyRegulation

401

The user is not authenticated with the OPERANDO system. Check that the service ticket provided by the authentication service is correctly included in the message body.

403

The user is authenticated with the OPERANDO system, but is not allowed to perform the requested action.

put /regulations/{reg-id}
Update an exisiting regulation. (regulationsRegIdPut)
Called by a regulator to ensure that the platform becomes compliant with the new terms of the regulation. The regulation that was saved is returned, along with an identifer which should be used to refer to it in later communication.

Path parameters

reg-id (required)
Path Parameter — the unique identifier of a regulation.

Request body

regulation body (required)
Body Parameter —

Return type

PrivacyRegulationInput

  • application/json

Responses

204

The regulation was successfully updated, and the platform has been made compliant with it. PrivacyRegulationInput

401

403

get /reports/{report-id}
Get a report. (reportsReportIdGet)
Called by a regulator to obtain a report matching the given ID. Intended (at the moment) to return a compliance report relating to an OSP.

Path parameters

report-id (required)
Path Parameter — the unique identifier of a report.

Query parameters

format (required)
Query Parameter — the requested format of the report (e.g. pdf, html)

Return type

String

Produces

This API call produces the following media types according to the Accept request header; the media type will be conveyed by the Content-Type response header.
  • application/json

Responses

200

The request was successful. The report is returned in the response body in the requested format. String

401

The user is not authenticated with the OPERANDO system. Check that the service ticket provided by the authentication service is correctly included in the message body.

403

The user is authenticated with the OPERANDO system, but is not allowed to perform the requested action.
