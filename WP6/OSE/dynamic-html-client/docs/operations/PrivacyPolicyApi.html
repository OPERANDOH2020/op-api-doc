Called by watchdog when it detects change in an OSPâ€™s privacy policy. OSE computes whether the OSP policy complies with regulations; complies with UPP. It updates UPP where appropriate and notifies users and OSP if there are issues with the updated privacy policy.\n\nPre-condition -- The OSP must be registered with OPERANDO and it must have an existing policy stored in the policy DB.

URL

HTTP Method

PUT

Response Type

Parameters

  • Path: osp_id BigDecimal

    The unqiue OPERANDO ID for the OSP.

  • Body: osp_policy List

privacyPolicyOspIdPolicyPost

Called by the management console when a new G2C OSP registers with OPERANDO. The new OSP tells the system its policies. Note G2C will be policy and data requests - hence when a new OSP registers it calls this method and the similar workflow method.

URL

http://operando.eu//PrivacyPolicy/{osp_id}/policy/

HTTP Method

POST

Response Type

Parameters

  • Path: osp_id BigDecimal

    The unqiue OPERANDO ID for the new OSP being registered.

  • Body: osp_policy List

    The initial policy entered when the OSP registers with OPERANDO

privacyPolicyOspIdWorkflowPost

Called by the management console when a new G2C OSP registers with OPERANDO. Tells the system its requests for data (Workflow). Note G2C will be policy and data requests; B2C will be settings.

URL

http://operando.eu//PrivacyPolicy/{osp_id}/workflow/

HTTP Method

POST

Response Type

Parameters

  • Path: osp_id BigDecimal

    The unqiue OPERANDO ID for the registered OSP service.

  • Body: osp_requests List

    The set of data requests that this service will make. Any requests not in this list, will be rejected by the OSP enforcement component of OPERANDO. G2C services must follow the information they provide here.
