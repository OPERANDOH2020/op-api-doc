This is the API

regulationsGet

The query contains a json object (names, values) and the request returns the list of documents (regulations) where the filter matches i.e. the document contains fields with those values.

URL

http://operando.eu/policyDB/Regulations/

HTTP Method

GET

Response Type

List

Parameters

  • Query: filter String

    The query filter to be matched - \"?filter=\"{json description}\"

regulationsRegIdDelete

Called when by the policy computation component when the regulator api requests that the regulation be deleted.

URL

http://operando.eu/policyDB/Regulations/{reg_id}/

HTTP Method

DELETE

Response Type

Parameters

  • Query: regulation_id BigDecimal

    Unique identifier representing a specific regulation

regulationsRegIdGet

Get a specific legislation document via the id. This will return the full legislation document in json format.

URL

http://operando.eu/policyDB/Regulations/{reg_id}/

HTTP Method

GET

Response Type

PrivacyRegulation

Parameters

  • Path: regulation_id BigDecimal

    Unique identifier representing a specific user

regulationsRegIdPost

Called by the policy computation component when a new regulation is added to OPERANDO.

URL

http://operando.eu/policyDB/Regulations/{reg_id}/

HTTP Method

POST

Response Type

Parameters

  • Path: regulation_id BigDecimal

    Unique identifier representing a specific user

  • Body: regulation PrivacyRegulation(PrivacyRegulation)

    The first instance of this regulation document

regulationsRegIdPut

Called when by the policy computation component when the regulator api updates a regulation. Their new PrivacyRegulation document is stored in the policy DB.

URL

http://operando.eu/policyDB/Regulations/{reg_id}/

HTTP Method

PUT

Response Type

Parameters

  • Path: regulation_id BigDecimal

    Unique identifier representing a specific regulation

  • Body: regulation PrivacyRegulation(PrivacyRegulation)

    The changed instance of this PrivacyRegulation
