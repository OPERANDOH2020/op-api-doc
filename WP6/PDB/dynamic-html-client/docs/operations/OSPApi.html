This is the API

oSPGet

The query contains a json object (names, values) and the request returns the list of documents (regulations) where the filter matches i.e. the document contains fields with those values.

URL

http://operando.eu/policyDB/OSP/

HTTP Method

GET

Response Type

List

Parameters

  • Query: filter String

    The query filter to be matched - \"?filter=\"{json description}\"

oSPOspIdDelete

Called when by the policy computation component when the regulator api requests that the regulation be deleted.

URL

http://operando.eu/policyDB/OSP/{osp_id}/

HTTP Method

DELETE

Response Type

Parameters

  • Query: osp_id BigDecimal

    Unique identifier representing a specific regulation

oSPOspIdGet

Get a specific OSP document via the id. This will return the full OSP document in json format.

URL

http://operando.eu/policyDB/OSP/{osp_id}/

HTTP Method

GET

Response Type

OSPPrivacyPolicy

Parameters

  • Path: osp_id BigDecimal

    Unique identifier representing a specific osp

oSPOspIdPost

Called by the policy computation component when a new regulation is added to OPERANDO.

URL

http://operando.eu/policyDB/OSP/{osp_id}/

HTTP Method

POST

Response Type

Parameters

  • Path: osp_id BigDecimal

    Unique identifier representing a specific osp

  • Body: regulation OSPPrivacyPolicy(OSPPrivacyPolicy)

    The first instance of this OSP document

oSPOspIdPut

Called when by the policy computation component when the regulator api updates a regulation. Their new OSPRequest document is stored in the policy DB.

URL

http://operando.eu/policyDB/OSP/{osp_id}/

HTTP Method

PUT

Response Type

Parameters

  • Path: osp_id BigDecimal

    Unique identifier representing a specific osp

  • Body: regulation OSPPrivacyPolicy(OSPPrivacyPolicy)

    The changed instance of this OSPRequest
